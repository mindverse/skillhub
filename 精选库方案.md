# 技能宝精选库方案

> 精选库策略：后台维护 3000-5000 个高质量 Skill，对外宣传"全量 35-50 万"。
> 用户日常使用主要触达精选库，体验好、安全有保障。

---

## 一、数据来源

### 收集渠道

| 渠道 | 预估Skill数量 | 质量 | 收录方式 |
|------|---------------|------|----------|
| **GitHub（主力）** | 30-50万+ | 参差不齐 | 自动爬取 + 筛选 |
| **awesome-claude-skills（ComposioHQ）** | 500+ | 较高 | 批量导入 |
| **awesome-claude-code（hesreallyhim）** | 300+ | 较高 | 批量导入 |
| **antigravity-awesome-skills** | 1000+ | 中等 | 批量导入筛选 |
| **Anthropic 官方 Plugins** | 13+ | 极高 | 手动导入 |
| **腾讯 Skill Hub** | 13000+ | 中等 | 参考复制 |
| **NPM / PyPI 生态** | 5000+ | 中等 | 自动爬取 |
| **社区提交** | 持续增长 | 需审核 | 人工审核 |
| **我们的卫星 Skill** | 50-100 | 高 | 直接入库 |

### 自动收录 vs 人工审核

```
全量库（35-50万）：
├── 自动收录 95%
│   ├── GitHub 爬虫：按 topic（claude-code-skill, claude-skill）自动发现
│   ├── 关键词搜索：搜索 "claude skill"、"claude code" 相关仓库
│   ├── 依赖分析：引用了 claude-code SDK 的项目
│   └── 自动评分：通过方案一基础评分自动打分
│
└── 人工审核 5%
    ├── 社区举报的 Skill
    ├── 评分异常的 Skill
    └── 被标记为可疑的 Skill

精选库（3000-5000）：
├── 自动筛选 60%
│   ├── 安全评分 >= 85
│   ├── GitHub Star >= 10 或下载量 >= 100
│   └── 近 90 天有更新
│
├── 人工精选 30%
│   ├── 编辑团队手动挑选优质 Skill
│   ├── 知名开发者/团队的作品
│   └── 填补分类空缺（某些类目空缺时主动寻找）
│
└── 卫星 Skill 10%
    └── 我们自己开发的高质量 Skill
```

### 爬取策略

**GitHub 爬取规则：**
```
1. 搜索条件：
   - topic: claude-code, claude-skill, claude-code-skill, claude-code-plugin
   - 文件名包含: .claude/, CLAUDE.md, skills/
   - 描述包含: "claude code skill", "claude skill"

2. 爬取频率：
   - 新 Skill 发现：每天一次
   - 已收录 Skill 更新检查：每周一次
   - 精选库 Skill 状态检查：每天一次

3. 去重规则：
   - 基于 GitHub 仓库唯一 ID
   - Fork 仓库默认不收录（除非有显著改进）
   - 同一作者的多版本取最新
```

## 三、精选策略

### 精选标准

一个 Skill 要进入精选库，需满足以下条件：

**必要条件（全部满足）：**
1. 安全评分 >= 80（B级及以上）
2. 有清晰的描述和使用文档
3. 功能可正常运行（非空壳项目）
4. 无恶意行为记录

**加分条件（满足越多排名越高）：**
| 条件 | 加分 |
|------|------|
| 安全评分 >= 95（S级） | +20 |
| GitHub Star >= 100 | +15 |
| GitHub Star >= 1000 | +30 |
| 知名团队/个人开发（如 Anthropic、Trail of Bits） | +25 |
| 近 30 天有更新 | +10 |
| 有中文文档或我们已汉化 | +15 |
| 属于热门分类 | +5 |
| 填补了分类空缺 | +10 |
| 是我们的卫星 Skill | +20（内部权重，不公开） |

### 从全量库筛选精选库的流程

```
全量库（35-50万）
    │
    ▼
[自动筛选] 安全评分 >= 80 + 有效 README + 近180天有活动
    │
    ▼
候选池（约 1-2 万）
    │
    ▼
[自动排序] 按加分条件计算综合分
    │
    ▼
Top 5000 候选
    │
    ▼
[人工审核] 编辑团队抽查 + 补充遗漏的优质 Skill
    │
    ▼
精选库（3000-5000）
    │
    ▼
[持续维护] 每周自动复查 + 每月人工更新
```

### 推荐排序算法

```
推荐分 = 基础分 × 时效因子 × 相关性因子 × 策略因子

基础分 = (
    安全评分 × 0.25 +
    质量评分 × 0.25 +          # Star、文档完整度、代码质量
    活跃度评分 × 0.15 +        # 更新频率、Issue 响应速度
    人气评分 × 0.20 +          # 下载量、安装量
    用户评价评分 × 0.15        # 用户评分、好评率
)

时效因子 = {
    最近7天更新:  1.2
    最近30天更新: 1.0
    最近90天更新: 0.9
    超过90天:     0.7
}

相关性因子 = {
    # 基于用户搜索关键词的匹配度
    标题完全匹配:  2.0
    标题部分匹配:  1.5
    描述匹配:      1.2
    标签匹配:      1.1
    无匹配:        1.0
}

策略因子 = {
    # 内部运营调控，不对外公开
    卫星 Skill:       1.5     # 我们的 Skill 优先
    合作开发者 Skill:  1.3     # 合作方优先
    新上架 Skill:      1.2     # 新 Skill 冷启动加权
    普通 Skill:        1.0
}
```

### 搜索排序规则

**用户搜索时的排序逻辑：**

```
1. 精选库 Skill 优先于非精选库
2. 精选库内按推荐分降序
3. 非精选库按基础分降序
4. 同分时按最近更新时间降序

特殊规则：
- 搜索"小红书"时，我们的"小红书"卫星 Skill 置顶
- 搜索结果前 3 个中至少包含 1 个我们的卫星 Skill（如有匹配）
- 安全评分 < 60 的 Skill 不出现在搜索结果中
```

## 四、精选库运营策略

### 冷启动阶段（第1-2周）

1. 从 awesome-claude-skills、awesome-claude-code 批量导入已验证的 Skill
2. 导入 Anthropic 官方 Plugins（13个）
3. 加入我们的首批卫星 Skill
4. 预计首批精选库：300-500 个

### 增长阶段（第3-8周）

1. 开启 GitHub 自动爬取，全量库快速增长到 10万+
2. 精选库通过自动筛选 + 人工审核增长到 1000-2000 个
3. 每周上线 2-3 个卫星 Skill
4. 开始接受社区提交

### 成熟阶段（第3个月+）

1. 全量库达到 35-50万（含长尾、低质量 Skill）
2. 精选库稳定在 3000-5000 个
3. 建立开发者生态，第三方主动提交
4. 每月更新精选库排名

### 数据展示策略

**对外展示（营销用）：**
- "技能宝收录 35万+ AI 技能"（全量库数字）
- "精选 3000+ 优质技能，每个都经过安全认证"

**对内管理：**
- 精选库 Skill 列表 + 实际使用数据
- 全量库仅作为搜索池，不保证质量

## 五、技术实现建议

### 数据存储

```
精选库索引文件：curated-skills.json（随客户端分发，离线可用）
全量库索引：线上数据库（SQLite / PostgreSQL）
Skill 详情：按需从 GitHub API 拉取
```

### 更新机制

```
客户端缓存策略：
├── 精选库索引：每天更新一次（约 500KB JSON）
├── 用户已安装的 Skill 信息：本地缓存
└── 搜索结果：实时查询线上 API

服务端更新策略：
├── GitHub 爬虫：每天凌晨运行
├── 安全评分重算：每周运行
├── 精选库重排：每周一次
└── 紧急下架：实时响应
```

### 索引文件大小控制

```
精选库 3000 个 Skill × 平均 500 字节/Skill ≈ 1.5 MB
压缩后约 300-500 KB，可随客户端分发
```
