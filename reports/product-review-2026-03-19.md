## 断裂点 ❌

### 1. skills.sh 搜索不到我们
- **用户感受**：在 skills.sh 搜 "skillhub" 或 "技能宝" 都找不到我们的 Skill。搜 "kevinaimonster" 也是 0 结果。用户通过 skills.sh 发现我们的可能性为零。
- **严重程度**：高
- **实际情况**：虽然 https://skills.sh/mindverse/skillhub 页面存在（200），但搜索索引似乎没收录我们。可能是安装量太少（<100）导致不在搜索结果中。
- **修复建议**：需要提升安装量才能进入 skills.sh 搜索结果。短期内在推广内容中直接给 GitHub 链接而不是依赖 skills.sh 搜索。

### 2. 默认安装只装 1 个 Skill
- **用户感受**：用户看到 README 说"50 个技能"，但执行简版命令 `npx skills add mindverse/skillhub -g -y` 后只装了 1 个（技能宝本体）。用户会困惑：说好的 50 个呢？
- **严重程度**：高
- **修复建议**：
  - README 中更醒目地区分"装全部"和"只装搜索引擎"
  - 或者：把 SKILL.md 从根目录移走，让默认安装就是 `--full-depth` 效果
  - 最简方案：README 只放全量安装命令，不给简版

### 3. 全量安装命令太长
- **用户感受**：`npx skills add mindverse/skillhub --full-depth --skill '*' -g -y` 这条命令 79 个字符，用户手动输入会打错，即使复制也可能漏掉引号。
- **严重程度**：中
- **修复建议**：考虑用短链接或脚本包装。但这是 skills CLI 的限制，短期无法改变。确保 README 中命令在代码块里，方便一键复制。

### 4. 安装完没有"下一步"引导
- **用户感受**：安装完成后 CLI 输出 "Done!"，但没告诉用户接下来做什么。用户不知道该对 AI 说什么来触发技能。
- **严重程度**：中
- **修复建议**：在 README "快速开始" 部分紧跟安装命令后面加使用示例。（已有，但不够醒目。可以加一个"安装完了？试试说这些"的引导框。）

### 5. 网站只展示 20 个 Skill（实际有 50 个）
- **用户感受**：网站 /skills 页面只展示了 skills-data.ts 里硬编码的 20 个 Skill，但实际 GitHub 有 50 个。用户看到的少了 30 个。
- **严重程度**：中
- **修复建议**：更新 dashboard-app 的 skills-data.ts 加入全部 50 个 Skill。或改为从 GitHub API 动态拉取。

### 6. 安全评分用户感知不到
- **用户感受**：安装过程中 CLI 显示了 "Safe / 2 alerts / Med Risk"——这是 skills.sh 自己的安全扫描。但我们说的"12 重安全检测"用户在哪里能看到？网站技能详情页有分数，但安装时看不到。
- **严重程度**：低
- **修复建议**：安全评分数据写入 SKILL.md 的 description 开头（如"安全评分 95/100 | S级"），这样 CLI 安装时就能显示。
